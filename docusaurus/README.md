## 特徴

- **Facebook（Meta）製**の React ベースドキュメントサイトジェネレータ
- **バージョン管理**が組み込み（v1/v2/v3 を並存できる）
- **全文検索**（Algolia DocSearch または `@easyops-cn/docusaurus-search-local`）
- MDX（Markdown + JSX）でリッチなコンポーネントを埋め込める
- サイドバー・ナビゲーションの設定が柔軟

## 向いている用途

- 複数バージョンを持つプロジェクトの仕様書
- 検索機能を重視する社内ポータル
- React チームが文書も管理したい場合

## セットアップ

```bash
cd docusaurus
npm install
npm run start       # http://localhost:3000 でプレビュー
npm run build       # build/ にビルド成果物が出力される
npm run serve       # ビルド後の確認用サーバー
```

### ローカル検索の有効化

```js
// docusaurus.config.js の themeConfig に追加
plugins: [
  [
    "@easyops-cn/docusaurus-search-local",
    {
      hashed: true,
      language: ["ja"],
    },
  ],
],
```

## 基本操作（SSG の作り方）

> 詳細は公式ドキュメント（[Docusaurus](https://docusaurus.io/docs)）を参照。ここでは最低限必要な操作だけまとめます。

### 記事（ページ）を追加する

1. `docs/` 配下に Markdown（`.md` / `.mdx`）を置く
2. サイドバーは `sidebars.js` で構成（`autogenerated` にすればフォルダ構成から自動生成）

ファイル先頭の **Front Matter** で表示順やタイトルを指定します。

```markdown
---
title: インストール手順
sidebar_position: 5
---

本文をここに書く。
```

### 内部リンクを作る

他ページへは **`.md` ファイルへの相対リンク** を書きます（ビルド時に検証され、リンク切れは `onBrokenLinks: 'throw'` でエラーになる）。

```markdown
詳しくは [開発ガイド](./getting-started.md) を参照。
```

### 画像・静的ファイルを管理する

- 静的ファイルは `static/` に置く（`static/img/foo.png` → `/img/foo.png` で参照）
- Markdown 内に置いた相対画像も Webpack がバンドルする

```markdown
![システム構成図](/img/architecture.png)
```

### ビルドとプレビュー

```bash
npm run start    # http://localhost:3000 でライブプレビュー
npm run build    # build/ に静的サイトを出力
npm run serve    # build/ をローカルサーバーで確認
```

## 配布方法のメリット・デメリット

### A. Web サーバーなしで HTML を直接配布する（file:// やファイル共有）

| | |
|---|---|
| ✅ | ビルド成果物（`build/`）は静的ファイルなので、Web サーバーに置けばどこでも動く |
| ❌ | **クライアントサイドルーティングの SPA** であり、出力は絶対パス前提。`file://` でダブルクリックしてもページ遷移・CSS が正しく動かない |
| ❌ | 全文検索もブラウザ上の JS が前提のため `file://` では機能しない |
| ⚠️ | どうしてもオフライン配布したい場合は、簡易 Web サーバー（`npx serve build` 等）ごと配るか、`trailingSlash` と相対 `baseUrl` を調整する必要がある |

> Docusaurus は「Web サーバーで配信する」前提のツールです。**サーバーなしの直接配布が主目的なら MkDocs / Sphinx が向いています。**

### B. GitLab Pages と連携する

`docusaurus.config.js` の `url` / `baseUrl` をサブパスに合わせます。

```js
// docusaurus.config.js
url: "https://<group>.gitlab.io",
baseUrl: "/<repo>/",   // プロジェクト配下に置く場合は必須
```

```yaml
# .gitlab-ci.yml
pages:
  image: node:20
  script:
    - cd docusaurus && npm ci && npm run build
    - mv build ../public
  artifacts:
    paths:
      - public
  rules:
    - if: $CI_COMMIT_BRANCH == $CI_DEFAULT_BRANCH
```

| | |
|---|---|
| ✅ | Web サーバー配信（GitLab Pages）なら検索・ルーティングが完全に動作する |
| ✅ | `baseUrl` を設定すればサブパス配信のリンクも自動で解決される |
| ❌ | `npm ci` + ビルドで CI 時間が長め（依存が多い） |
| ❌ | `url` / `baseUrl` の設定漏れがあるとリンク切れ（`onBrokenLinks: throw`）でビルドが失敗する |

## 長所 / 短所

| | |
|---|---|
| ✅ | バージョン管理が標準搭載 |
| ✅ | MDX で React コンポーネントを埋め込める |
| ✅ | 全文検索・タグ・ブログも対応 |
| ❌ | Node.js / React の知識が前提 |
| ❌ | 初回 `npm install` が重い（依存が多い） |
| ❌ | ビルド速度は MkDocs / Hugo より遅い |
