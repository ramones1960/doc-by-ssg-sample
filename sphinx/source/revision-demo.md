# 改訂マーク デモ

このページでは `revision` ディレクティブの使い方と表示例を示します。

## 使い方

### MyST Markdown（`.md` ファイル）

````md
```{revision} A
A版で改訂した内容をここに書きます。
```
````

コロンフェンス記法でも書けます。

```md
:::revision A
A版で改訂した内容をここに書きます。
:::
```

### reStructuredText（`.rst` ファイル）

```rst
.. revision:: A

   A版で改訂した内容をここに書きます。
```

---

## 表示例

通常テキスト（改訂なし）はそのまま表示されます。

```{revision} A
この段落は **A版** で改訂されました。
右端の縦線と "A" ラベルが改訂箇所を示しています。
```

改訂がない段落はマークなしで表示されます。

```{revision} B
仕様変更により、処理タイムアウトを **30 秒** から **60 秒** に延長しました。
また、リトライ回数の上限を 3 回から 5 回に変更しています。
```

---

## 表の一部が改訂された場合の例

| 項目         | 旧仕様     | 新仕様     |
|------------|----------|----------|
| タイムアウト     | 30 秒     | 60 秒     |
| リトライ上限     | 3 回      | 5 回      |

```{revision} B
上記の表はB版で更新されました。タイムアウトとリトライ上限が変更されています。
```

---

## 複数の版が混在する例

```{revision} A
A版：初版リリース時に追加されたセクションです。
```

通常のテキストが続きます。

```{revision} B
B版：要件変更を受けて内容を更新しました。
```

さらに通常のテキストが続きます。

```{revision} C
C版：レビュー指摘を反映し、表現を修正しました。
```
